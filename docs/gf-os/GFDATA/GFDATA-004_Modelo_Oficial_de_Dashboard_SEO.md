## 1. Finalidade

O documento **GFDATA-004 - Modelo Oficial de Dashboard SEO do GetEstimateFast** define o padrão oficial para acompanhamento, análise e tomada de decisão sobre SEO orgânico da plataforma.

Seu objetivo é transformar dados de busca em inteligência prática para crescimento orgânico, priorização de páginas, melhoria de conteúdo, expansão local e geração de leads qualificados.

Este dashboard deve conectar métricas de SEO com a estratégia de aquisição, produto, conversão e expansão do GetEstimateFast.

---

## 2. Objetivo Central

O Dashboard SEO deve responder uma pergunta principal:

**Quais páginas, serviços, cidades e clusters orgânicos estão gerando visibilidade, tráfego e oportunidades reais de lead para o GetEstimateFast?**

Ele deve permitir identificar:

- páginas que crescem;
- páginas que caem;
- keywords com oportunidade;
- clusters com demanda;
- serviços com potencial orgânico;
- cidades com tração;
- conteúdos que precisam de atualização;
- páginas com impressões, mas baixo CTR;
- páginas com tráfego, mas baixa conversão.

---

## 3. Fontes de Dados

As fontes principais do Dashboard SEO serão:

- Google Search Console;
- Google Analytics 4;
- dados internos de conversão;
- dados de leads;
- auditorias técnicas;
- ferramentas externas de SEO, quando utilizadas;
- pesquisas de demanda local;
- relatórios manuais de SERP.

O Google Search Console será a fonte principal para impressões, cliques, CTR, posição média, queries e páginas orgânicas.

## 6. Análise por Página

Toda página orgânica relevante deve ser analisada individualmente.

### 6.1 Métricas por página

| Métrica | Uso |
|---|---|
| URL | Identificar a página |
| Tipo de página | Serviço, cidade, blog, guia ou landing page |
| Impressões | Medir visibilidade |
| Cliques | Medir tráfego |
| CTR | Medir atratividade |
| Posição média | Medir ranqueamento |
| Conversões | Medir resultado |
| Leads | Medir valor |
| Status | Crescendo, estável, caindo ou crítica |

### 6.2 Classificação da página

Cada página pode receber uma classificação:

| Status | Significado |
|---|---|
| Crescendo | Aumento relevante de impressões, cliques ou conversões |
| Estável | Sem variação relevante |
| Oportunidade | Muitas impressões, poucos cliques |
| Fraca | Poucas impressões e poucos cliques |
| Crítica | Queda forte ou perda de posição |
| Alta conversão | Pouco tráfego, mas bom resultado |
| Baixa conversão | Tráfego sem leads |

## 11. Oportunidades de CTR

Uma das funções mais importantes do Dashboard SEO é identificar páginas com muitas impressões e poucos cliques.

### 11.1 Critério inicial

Uma página ou query pode ser marcada como oportunidade de CTR quando tiver:

- impressões relevantes;
- posição média razoável;
- CTR abaixo do esperado;
- título pouco atrativo;
- meta description genérica;
- intenção de busca não respondida claramente.

### 11.2 Ações recomendadas

- melhorar title tag;
- melhorar meta description;
- adicionar termos locais;
- deixar o benefício mais claro;
- incluir serviço e cidade;
- reforçar confiança;
- alinhar conteúdo com a intenção da query.

---

## 12. Oportunidades de Conteúdo

O Dashboard SEO deve apontar lacunas de conteúdo.

Uma oportunidade de conteúdo pode surgir quando:

- uma query tem impressões, mas não existe página específica;
- uma cidade aparece em buscas, mas não possui página dedicada;
- um serviço tem demanda, mas pouca cobertura;
- um artigo ranqueia para termos transacionais;
- concorrentes possuem páginas mais específicas;
- o usuário busca informação regulatória ou local ainda não respondida.

### 12.1 Tipos de conteúdo recomendados

- página de serviço;
- página local;
- página serviço + cidade;
- guia educativo;
- artigo de blog;
- FAQ local;
- checklist;
- comparativo;
- página de compliance;
- página de storm preparedness.

---

## 13. Oportunidades de Conversão

SEO não deve ser medido apenas por tráfego.

O dashboard deve identificar páginas com tráfego orgânico, mas baixa conversão.

Possíveis causas:

- CTA fraco;
- formulário distante;
- falta de confiança;
- página muito informacional;
- falta de prova local;
- falta de promessa clara;
- conteúdo desalinhado com intenção;
- problema técnico em mobile;
- velocidade baixa;
- falta de telefone ou contato direto.

Ação recomendada: enviar a página para revisão conjunta com GFSEO, GFPROD e GFDATA.

## 17. Relação com GFSEO

O Dashboard SEO deve se conectar diretamente com a série GFSEO.

Ele mede se a estratégia documentada em GFSEO está funcionando.

A relação prática é:

- GFSEO define estratégia;
- GFDATA mede desempenho;
- GFPROD melhora conversão;
- GFENG garante implementação técnica;
- GFOPS informa qualidade real dos leads;
- GESMP orienta decisão estratégica.

---

## 18. Relação com Produto

Quando uma página tem tráfego, mas não converte, o problema pode não ser SEO.

Pode ser:

- experiência ruim;
- CTA pouco claro;
- fluxo longo;
- baixa confiança;
- página lenta;
- layout ruim no mobile;
- formulário inadequado.

Por isso, o Dashboard SEO deve sinalizar páginas que precisam de revisão de UX e conversão.

---

## 19. Relação com Operações

A operação deve informar se os leads orgânicos são bons ou ruins.

O Dashboard SEO deve buscar responder:

- SEO está trazendo leads válidos?
- quais serviços orgânicos geram melhores leads?
- quais cidades geram leads aproveitáveis?
- quais páginas atraem usuários fora da área?
- quais conteúdos geram leads fora do escopo?

Essa retroalimentação evita crescimento orgânico sem valor comercial.

---

## 20. Critério de Sucesso

O Dashboard SEO será considerado bem-sucedido quando permitir responder:

- quais páginas orgânicas geram tráfego;
- quais páginas orgânicas geram leads;
- quais serviços têm maior potencial orgânico;
- quais cidades devem ser priorizadas;
- quais queries merecem novas páginas;
- quais páginas precisam de melhoria;
- quais conteúdos devem ser atualizados;
- quais quedas exigem ação imediata;
- quais clusters estão fortalecendo o GetEstimateFast.

---

## 21. Conclusão

O **GFDATA-004** define o modelo oficial de Dashboard SEO do GetEstimateFast.

Ele transforma dados de busca em decisões práticas para crescimento orgânico, geração de leads e expansão local.

Com este dashboard, o GetEstimateFast poderá acompanhar melhor sua visibilidade no Google, identificar oportunidades antes dos concorrentes, melhorar páginas existentes, criar novos conteúdos com base em demanda real e conectar SEO diretamente ao resultado do negócio.
